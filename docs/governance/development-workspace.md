# Agile AI Development Workspace

## Purpose

This document explains the Visual Studio Code workspace configuration used for the Agile AI ecosystem.

The workspace allows the core repositories of the initiative to be opened together in a single development environment while maintaining a clear separation between research, standards development, and operational systems.

---

# Workspace Configuration

The Agile AI development environment uses a Visual Studio Code workspace file:

AgileAI-University.code-workspace

This file is **not source code**.  
It is a configuration file that tells Visual Studio Code which repositories should be opened together.

The workspace file is stored in the following repository:

FB-AgileAI-University

---

# What the Workspace File Loads

When the workspace file is opened, Visual Studio Code automatically loads the following repositories:

FB-AgileAI-University  
agileai-foundation  
agileai-research

This allows the entire Agile AI ecosystem to be viewed and developed within a single workspace.

---

# Opening the Workspace

To open the complete development environment:

1. Locate the file:

AgileAI-University.code-workspace

2. Double-click the file.

Visual Studio Code will open with the following structure:

AGILEAI-UNIVERSITY (WORKSPACE)

FB-AgileAI-University  
agileai-foundation  
agileai-research

No additional setup is required.

---

# Ecosystem Repository Architecture

The Agile AI ecosystem is organized into three layers.

---

## Research Layer

Repository:

agileai-research

Purpose:

Private repository used for:

- exploratory ideas
- experimental architectures
- early conceptual drafts
- internal research notes

Content in this repository **does not automatically represent the official Agile AI reference model**.

---

## Specification Layer

Repository:

agileai-foundation

Purpose:

Public repository containing:

- official concepts
- architectural explanations
- reference documentation
- structural models

This repository represents the **public reference body of knowledge for Agile AI**.

---

## Education and Adoption Layer

Repository:

FB-AgileAI-University

Purpose:

Operational systems supporting:

- Agile AI learning ecosystem
- capability assessments
- credential infrastructure
- portal services

This layer enables **education, professional recognition, and ecosystem adoption**.

---

# Concept Lifecycle

The repository structure reflects the lifecycle of ideas in the Agile AI ecosystem.

Research  
↓  
Specification  
↓  
Education / Adoption

Or in repository terms:

agileai-research  
↓  
agileai-foundation  
↓  
FB-AgileAI-University

---

# Research Disclaimer

The `agileai-research` repository contains exploratory work.

Materials in this repository may include:

- experimental models
- early architecture concepts
- conceptual drafts

These materials are **not part of the official Agile AI reference model** until they are formally incorporated into the `agileai-foundation` repository.

---

# Institutional Rationale

Separating repositories into research, specification, and adoption layers ensures:

- architectural clarity
- safe experimentation
- stability of the public reference model
- controlled evolution of concepts

This structure aligns with practices used by many large technology ecosystems, including standards bodies, research initiatives, and open technical foundations.
