Agile AI Institutional Print Constitution v1.0

Issued by the Agile AI Foundation as the canonical print and publication standard governing all institutional PDF artifacts.

1. Authority

This Print Constitution defines mandatory structural, typographic, and layout standards for all constitutional and interpretive artifacts issued under Agile AI Foundation authority.

No artifact may deviate from this specification without formal version advancement of this Print Constitution.

2. Page Architecture

All institutional print artifacts shall use:

  • A4 format
  • Margins: 28mm (top & bottom), 25mm (left & right)
  • No decorative borders or visual embellishments

3. Typography System

Body text shall use a serif typeface (Georgia or Times New Roman) at 12pt with 1.45 line-height.

Primary and secondary headings shall use a clean sans-serif system font.

Typography shall remain restrained and publication-grade.

4. Hierarchy Discipline

Constitutional artifacts (e.g., Functional Elements) shall remain compact and declarative.

Interpretive artifacts (e.g., Guides) may use sectional pagination but shall not alter typographic hierarchy.

5. Footer & Canonical References

Canonical authority shall be declared on the cover page.

Repeating footers are discouraged unless required for legal preservation.

6. Version Governance

Each artifact must clearly display:

  • Version number
  • Reference ID
  • Issued year
  • Canonical source URL

No silent modification is permitted. All structural or layout revisions require version increment and archival preservation.

7. Integrity Principle

Institutional integrity is achieved through constraint, consistency, and governance. Aesthetic variation shall never supersede structural coherence.
