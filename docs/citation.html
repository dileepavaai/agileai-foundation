AGILE AI FOUNDATION — REFERENCE INTEGRITY PROTOCOL

Citation & Reference Guidelines

This page defines the official citation protocol for Agile AI Functional Elements. Stable referencing ensures institutional clarity, version discipline, and preservation of canonical integrity.

1. Stable Canonical Citation (Active Standard)

The official stable citation endpoint for the active canonical standard is:

https://agileai.foundation/standards/current/

This endpoint always resolves to the currently active canonical version. It should be used in academic, professional, and institutional references unless a specific historical version is required.

Suggested Citation (Active Canonical)

Agile AI Foundation. (2026). Agile AI Functional Elements (Version 1.1). Agile AI Foundation. https://agileai.foundation/standards/current/

2. Version-Specific Citation

When referencing a specific historical version, citations must include the version number and reference the corresponding versioned path.

Example (Version 1.1):

https://agileai.foundation/standards/v1.1/

Suggested Citation (Version-Specific)

Agile AI Foundation. (2026). Agile AI Functional Elements (Version 1.1). Agile AI Foundation. https://agileai.foundation/standards/v1.1/

Archived versions remain permanently accessible and citable. Canonical advancement occurs only through formal version increment and archival publication.

3. Canonical Identity

Canonical Reference ID: AAF-FAE-1.1-2026

The Canonical Reference ID uniquely identifies the constitutional release of the Functional Elements. Future versions will receive distinct reference IDs.

4. Internal Linking Discipline

  • Institutional surfaces should link to /standards/current/ when referencing the active canonical standard.
  • Versioned links (e.g., /standards/v1.1/) should be used only when referencing a specific historical baseline.
  • The Governance Index defines formal version advancement and archival control.
